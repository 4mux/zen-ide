## Overview

Zen IDE needs a full Vim modal editing experience so keyboard-only users can navigate and edit code without touching the mouse — matching the core Vim workflow (Normal, Insert, Visual, Command-line modes with motions, operators, text objects, registers, search, macros, etc.).

## Approach: GtkSourceView 5's Built-in `GtkSource.VimIMContext`

GtkSourceView 5 ships a fully-featured Vim emulation layer via `GtkSource.VimIMContext`. This provides:

- All standard modes: Normal, Insert, Replace, Visual, Visual Line, Visual Block
- Motions: `h/j/k/l`, `w/b/e/W/B/E`, `0/$`, `gg/G`, `{/}`, `f/F/t/T`, `%`, etc.
- Operators: `d`, `c`, `y`, `>`, `<`, `=`, `gq`, `~`, `gu`, `gU`
- Text objects: `iw/aw`, `i"/a"`, `i(/a(`, `it/at`, etc.
- Command-line: `:w`, `:q`, `:wq`, `:%s/foo/bar/g`, `:set`, `:!command`, etc.
- Registers, marks, `.` repeat, macros (`q`), counts, undo/redo
- Search: `/pattern`, `?pattern`, `n`, `N`, `*`, `#`

We **don't** build a Vim state machine from scratch. Instead, we integrate the native GTK widget, wire up signals for mode changes + command execution, and handle Zen-specific commands (`:w` → save, `:q` → close tab, etc.).

## Implementation Phases

### Phase 1: Core Vim Engine Integration

#### 1. VimModeManager (`src/editor/vim_mode.py`)
- Instantiate `GtkSource.VimIMContext()`
- Connect it to the `GtkSource.View` via `set_client_widget()`
- Wire `EventControllerKey` to route keys through `filter_keypress()`
- Connect `notify::command-bar-text`, `notify::command-text`, `notify::mode` signals
- Connect `execute-command` signal for custom command routing
- Provide `enable()`/`disable()` toggle methods
- Handle cursor shape: block cursor in Normal mode, line cursor in Insert mode

#### 2. VimCommandRouter (`src/editor/vim_command_router.py`)
- Route `:w` → file save (existing `EditorTab.save()`)
- Route `:q` → close tab (existing `EditorView.close_tab()`)
- Route `:wq` / `:x` → save + close
- Route `:q!` → close without saving
- Route `:e <path>` → open file
- Route `:tabnew` → new tab
- Route `:tabnext` / `:tabprev` → tab navigation
- Route `:noh` → clear search highlights
- Fallback to VimIMContext default for unrecognized commands

#### 3. Editor Integration (`src/editor/editor_view.py`)
- In `EditorTab.__init__()`: create `VimModeManager` instance per tab
- In `EditorTab._on_key_pressed()`: delegate to vim manager (after inline completion, before autocomplete)
- Gated by `get_setting("behavior.is_nvim_emulation_enabled")`
- Ensure vim mode does NOT activate when terminal or tree has focus

#### 4. Status Bar Mode Display (`src/main/status_bar.py`)
- Unhide `self._mode_label` when vim mode is enabled
- Add `set_vim_mode(mode: str)` method
- Add `set_command_text(text: str)` method for partial commands / command-line
- Color-code mode label (e.g., green=NORMAL, blue=INSERT, orange=VISUAL)
- Add command bar text display (for `:`, `/`, `?` input)

### Phase 2: Settings & Polish

#### 5. Settings Integration
- Extend `default_settings.py` with vim-specific defaults under `behavior`
- Ensure `is_nvim_emulation_enabled` controls the full vim mode (not just popup style)
- Add runtime toggle: enable/disable vim mode without restart

#### 6. Cursor Shape Switching
- Normal mode: block cursor (overwrite style)
- Insert mode: thin line cursor (standard)
- Visual mode: block cursor with selection highlighting
- Replace mode: underline cursor
- Leverage GtkSourceView's `set_overwrite()` and CSS for cursor styling

#### 7. Keybinding Harmony
- In Normal mode: `Ctrl+C` should be vim escape (not clipboard copy) — VimIMContext handles this
- Zen shortcuts that should ALWAYS work regardless of mode: `Cmd+S`, `Cmd+P`, `Cmd+Shift+P`, `Cmd+,`
- Zen shortcuts that vim should override in Normal mode: arrow keys, Ctrl+A/E (line start/end)
- Document which Zen shortcuts are vim-overridden vs preserved

### Phase 3: Enhanced Features

#### 8. Visual Feedback
- Flash/animate mode transitions in status bar
- Highlight all search matches when using `/` or `*`
- Show search count in status bar ("3/17 matches")
- Relative line numbers option when in Normal mode

#### 9. Command-line Popup UI (Optional)
- Show `:` commands in a popup anchored to status bar area (like real Vim)
- Inherits from `NvimPopup`
- Tab completion for commands and file paths
- Command history (up/down arrow)

## Notes & Considerations

- **GtkSource.VimIMContext** does the heavy lifting — it implements ~90% of vim behavior natively in C, so it's fast and correct. We mostly wire signals and handle Zen-specific commands.
- **Terminal isolation** is critical — vim mode must NEVER interfere with terminal input. The focus manager gates this.
- **Inline AI completion** takes priority over vim keys — if ghost text is showing, Tab/Enter accept it before vim processes the key.
- **Per-tab state** — each editor tab gets its own VimModeManager, but mode changes propagate to the shared status bar.
- **Performance** — VimIMContext is native C code, so no Python overhead for key processing. Mode changes are signal-driven, not polled.
- **Graceful fallback** — if VimIMContext is unavailable (old GtkSourceView), log a warning and disable vim mode silently.
- **`:set` bridge** — VimIMContext supports `:set number`, `:set wrap`, etc. We can bridge these to Zen settings where applicable.
